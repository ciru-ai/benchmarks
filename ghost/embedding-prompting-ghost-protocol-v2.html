Prompted embedding strategy

Prompting decides what the ghost learns.

Embeddings do not arrive with our review categories baked in. A prompt is the contract that tells a model whether similarity should mean shared topic, shared emotional tone, shared risk, shared conversational role, or shared deletion usefulness. For Ghost Protocol, that distinction is the difference between a generic tweet map and a usable training loop.

Why Prompting Matters

Coordinates need intent

Embeddings are coordinates, not categories

A small embedder places each tweet in a vector space. Clusters appear because nearby tweets share some pattern, but the model still needs instruction about which patterns matter for review.

The prompt changes what gets preserved

If the instruction says "semantic clustering," the model should retain topic, sentiment, intent, and conversational role. If the instruction is absent or wrong, it may overvalue names, URLs, repeated phrasing, or surface words.

Ghost Protocol needs decision similarity

Our useful unit is not "tweets about the same word." It is "tweets likely to need the same review decision." The prompt should steer the map toward why an operator would keep, delete, defer, or create a rule.

Prompting Rules

What we can control

Use native controls before text prompts

Jina's correct setup is its native clustering task. That is already a form of model-aware prompting, and stacking our own text instruction on top made the output worse.

Keep Qwen instructions short and operational

Qwen improved when the prompt named the task and the review dimensions. The short prompt was enough: it told the model what to preserve without turning every tweet into a verbose task description.

Measure cluster shape, not just scores

PPLX looked better on Davies-Bouldin after prompting, but its median cluster size became 1 and one cluster swallowed more than half the sample. That is not useful for board-level triage.

Treat prompts as product settings

The prompt is not a lab detail. It encodes the review objective. For this product, the right instruction should optimize for human review decisions, not generic benchmark similarity.

How We Apply This To Ghost Protocol

Training loop

Operating principle

Ghost Protocol should use embeddings to propose lanes for review, then use human decisions to learn the boundary. The model can surface neighborhoods. The operator decides what those neighborhoods mean.

Build a mixed sample Blend flagged tweets, random unflagged tweets, high-engagement items, low-engagement items, old posts, replies, quotes, and edge cases. Mixed signal teaches both delete and leave-alone boundaries.
Run two embedding lanes Use Jina task-only as the discovery lane and prompted Qwen 0.6B as the open local comparison lane. Do not spend more time on heavier models until the human loop proves what extra quality would buy.
Generate candidate review lanes Cluster into practical groups such as personal disclosure, emotional outbursts, dated reactions, support complaints, routine chatter, dead-link posts, low-upside replies, and high-value keep candidates.
Present clusters as hypotheses The UI should not say "this category is true." It should say "this group may share a review reason." Ghost Training then samples examples from inside and outside that group.
Capture keep, delete, defer, and reason Each operator decision becomes training signal. Positives and negatives are both valuable because the ghost is learning boundaries, not merely collecting examples of bad tweets.
Promote stable patterns into directives Only after repeated agreement should a cluster become a named subcategory, a detector candidate, or a Ghost Directive. The directive stays reviewable and auditable.

Prompt We Should Use

Qwen lane

For the Qwen lane, use a short instruction that names the review dimensions without inventing final categories. This is intentionally not a delete prompt. It is a clustering prompt for Ghost Protocol review. 

Represent this tweet for semantic clustering. Group tweets by topic, sentiment, intent, and conversational role; ignore usernames and URLs.

Topic

Captures what the tweet is about, but does not let topic alone decide review action.

Sentiment and intent

Separates complaint, joke, confession, reaction, promotion, apology, argument, and routine update.

Conversational role

Distinguishes reply, quote reaction, announcement, personal note, support request, and social filler.
