Ghost Protocol Embedding Bench

Three small embedders sorted a 100,000-tweet sample.

We asked each model to do the same job: map messy short-form social text into neighborhoods, then let clustering reveal what kinds of review lanes might exist. The result is not a final product decision, but it is a practical signal for speed, quality, and licensing tradeoffs.

The Story

Think of embeddings as asking each model to arrange tweets on a giant table. Similar tweets should end up near each other. Clustering then draws circles around nearby piles. The important question is not just speed. It is whether the piles have names a human reviewer can use.

Qwen 0.6B

Qwen was the fastest sustained encoder by a small margin. It found recognizable neighborhoods, but the k=12 cluster map still produced a very large reply-and-gratitude pile plus one tiny residual cluster.

PPLX 0.6B

PPLX matched Qwen on sustained speed while using the least memory. Its unsupervised clusters were broad and its silhouette landed near zero, which makes it better suited to seeded similarity search than open-ended discovery.

1,502 tweets/sec

Jina v5 small

Jina acted more like a librarian. It was slower, but it produced the strongest separation, the highest mean centroid similarity, and the healthiest minimum cluster size. That is the behavior we want for first-pass discovery.

723 tweets/sec

Speed Versus Shape

The 100,000-tweet run used batch size 64 on an RTX 4080 SUPER. All three models fit comfortably in 16 GB VRAM. The board decision should weigh sustained speed against whether the clusters reduce human review effort.

Cluster Field Notes

These are not labels created by the models. They are our human labels after inspecting cluster terms and representative tweets. This distinction matters: the embedder discovers neighborhoods; the product team decides whether a neighborhood is useful.

Cluster Balance

A useful sorter should avoid tiny orphan clusters and one giant bucket that eats everything. At k=12, Jina had the strongest minimum cluster size: 4,333 tweets. Qwen had a 10-tweet residual cluster, and PPLX had a 452-tweet smallest cluster.

Decision Table

The board choice is not one-dimensional. The right answer changes depending on whether we are optimizing for discovery quality, legal deployability, speed, or a future production path.

Question Qwen 0.6B PPLX 0.6B Jina v5 small
Best use case General deployable baseline with strong open-model pedigree. High-speed similarity search and cost-sensitive full-archive passes. Unsupervised topic discovery and board-readable cluster exploration.
What it did well Fastest sustained throughput in this run and a permissive model path. Near-Qwen speed with the lowest observed VRAM footprint. Best separation, strongest centroid similarity, and healthiest cluster balance.
What to watch Clusters can become broad social chatter buckets, and one k=12 cluster collapsed to only 10 tweets. Near-zero silhouette suggests open-ended discovery needs more guidance. Non-commercial license caveat and slower throughput.
Board-level interpretation Strong deployable default if we value speed, licensing, and acceptable baseline quality. Best low-memory candidate if we already know the categories we want. Best discovery candidate if we are still learning the archive's natural lanes.

Recommendation

Use a two-lane strategy. Use Jina v5 small as the discovery benchmark because it produced the strongest natural clusters. Use Qwen 0.6B or PPLX 0.6B as the deployable candidate once categories are seeded and licensing matters.

Do not jump to Qwen 8B yet. The small models are already fast enough to test the product workflow at 100,000-tweet scale. The next constraint is not model size. It is whether the category loop improves human review decisions.

Next Experiment

Name natural clusters.

Use Jina to surface candidate piles, then manually assign board-friendly names like low-upside replies, personal routine, tech support, dead weight, or strong keep.

Create seeded categories.

Pick 20 positive and 20 negative examples per category. Use prototype similarity to find the next 200 likely matches.

Measure seeded precision and recall.

Unsupervised clustering favors Jina. Seeded classification may favor the faster permissive models.

Promote only explainable rules.

Embeddings should suggest review lanes, not auto-delete. Durable criteria should become explicit detectors or auditable category thresholds.

Important caveat: This benchmark used 100,000 records sampled from a generic 1.6M tweet sentiment CSV, not Brad's final X archive format. The run is valid for local speed, memory behavior, and model behavior on messy social text, but it should not be treated as final domain performance. The product parser should continue to target canonical tweet records, not this CSV shape.
