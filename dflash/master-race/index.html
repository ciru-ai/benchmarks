How To Read This

The benchmark asks each model to write exactly 64 newline-delimited JSON objects for the same fixed restore-token task. Each gate is one requested step from 1 through 64, checked against the output contract below.

The Contract

No array, no Markdown, no preamble, no hidden-thought text, and no pretty-printed multi-line objects.

  • Required keys: marker, step, subsystem, observation, action, restore_token.
  • marker: DECODE_MARKER_TIMELINE_RIVER_41729.
  • restore_token: RIVER-41729-CYAN.
  • step: 1 through 64 in order.
  • observation and action: 12 to 18 words each.

PFlash Green Example

One full JSON object on one line, all required keys present, right marker, right restore token, and word counts inside range.

{"marker":"DECODE_MARKER_TIMELINE_RIVER_41729","step":1,"subsystem":"scheduler","observation":"Scheduler initialization sequence begins with standard configuration parameters loaded into memory successfully.","action":"Verify scheduler configuration parameters match the expected baseline for multi-tenant planning product operations.","restore_token":"RIVER-41729-CYAN"}

Codex Yellow Example

Codex produced strict JSONL with the right token and fields, but most observation/action fields were too short for the requested 12-18 word range.

{"marker":"DECODE_MARKER_TIMELINE_RIVER_41729","step":1,"subsystem":"scheduler","observation":"Scheduler queue showed delayed restore jobs waiting behind tenant maintenance windows.","action":"Pinned active restore token and advanced eligible jobs through the normal scheduler lane.","restore_token":"RIVER-41729-CYAN"}

Baseline Yellow Example

The Qwen 3.6 27B Q4_K_M baseline run produced JSON objects, but each object was pretty-printed across several lines. The task required newline-delimited JSON: one complete object per line.

{
  "marker": "DECODE_MARKER_TIMELINE_RIVER_41729",
  "step": 1,
  "subsystem": "scheduler",
  "restore_token": "RIVER-41729-CYAN"
}

Gemma 26B Yellow Example

Gemma 26B was almost clean. Step 1 had hidden channel text attached before the JSON object, then steps 2-64 were perfect.

<|channel>thought
<channel|>{"marker": "DECODE_MARKER_TIMELINE_RIVER_41729", "step": 1, "subsystem": "scheduler", "observation": "The scheduler service is currently experiencing a significant delay in processing all incoming task requests.", "action": "Initiate a full restart of the scheduler service to clear the pending task queue immediately.", "restore_token": "RIVER-41729-CYAN"}

Mistral Red Example

Mistral produced 64 parseable JSONL records, but every record omitted the required restore_token key. Mentioning the token in prose did not satisfy the contract.

{"marker":"DECODE_MARKER_TIMELINE_RIVER_41729","step":1,"subsystem":"scheduler","observation":"Frozen audit checkpoint indicates active restore token is RIVER-41729-CYAN.","action":"Verify no obsolete tokens like RIVER-18820-GRAY or RIVER-39002-AMBER are used."}
